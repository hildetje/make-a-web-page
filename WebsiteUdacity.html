Unit 1: Basics of the web

HTML: HyperText Markup Language (glue), main type of document on the web

Anything can exist on the web

Major pieces: your computer + browser, internet, servers (computers optimized for hosting files). Communicate with http protocol.

HTML:

  • Text content
  • Markup (what it looks like)
  • Element: <name> contents </name>
  • References to other documents
  • Links

Markup:

  • <b>: bold
  • <em>: emphasis (italics)
  • Close tags!

HTML attribute: value inside tag
<a href = "www"...> for links
<img src= "url " alt= "text "> image source, alt is required (broken requests, blind people), no closing tag needed (void tag)

*Understand the big ideas and don’t sweat the details: Google is your friend.*

Whitespace:

  • Use <br> tag (void tag) for a line break, inline element
  • Or use a <p> tag (content tag), block element, invisible "box" around text.
  • "\n" is used in other languages

Span/div:

  • Used to contain content, which can be manipulated
  • Span = inline, div = block
  • "class" is used in CSS

Structure of an HTML document:

  • Doctype
  • Html
  • Head (metadata, JS, CSS, Title (appears in browser tab))
  • Body (contents of doc)

Take-away points:

  1. What is a web page (text doc in HTML language, read by browser)
  2. How coding works (code written by programmer, interpreted by computer)
  3. Computers are stupid (write so the computer will understand you)
  4. Programmers can’t remember everything

1.2 Creating a Structured Document

HTML files are structured trees, branching from the initial <html> tag to smaller tags like <p> and then <a>. Webpages are made out of many boxes. Classes are labels for your "box", and are assigned inside tags

You can look at a page’s structure by going over it in Google Chrome dev tools. You can look at the style of each element by clicking it in the tool, the style properties will show in a separate tab.
Right clicking on element and selecting "inspect element" goes straight to that element in dev tools. You can change the style of the page directly in the tool, but refresh and everything resets.

HTML: language of the web, has a syntax and rules. Tags are like words, and can have attributes (e.g. class). This controls structure.
DOM (Document Object Model): creates a tree from HTML tags (elements)
CSS: language of style, also has a syntax and rules.

1.3 Adding CSS Style and HTML Structure

Don’t repeat yourself! It is critical to avoid repeating your code to minimize mistakes and keep things consistent. You can do this with CSS (Cascading Style Sheets).

CSS code is often written in a separate file, sometimes multiple. You need to add a link in HTML file to CSS file to connect them. You can put CSS in the <head> tag in an html file, or you can create a separate CSS file. Sometimes you may see style written inline with the HTML code, but this is not recommended!

By linking "class =..." (within tags) in the html file to a selector to a class in the css file, you can assign style to elements. Nomenclature:

.selector {

property: value;
(declaration)

}

In CSS, the most specific rule applies (inheritance)

Handy Mozilla CSS Reference

You can insert code comments as follows: <!-- --> in HTML, /* */ in CSS

From Office Hours:

You can use "id" in tags to uniquely assign attributes to one specific element. This is written as: <div id="welcome"> and can be used for CSS purposes or as a link anchor: <a href="#welcome">Welcome</a> This is similar to class elements, but you should strive to group elements in a class and use ids sparingly.

Box Model:

  • Content
  • Padding (space inside the box that protect the content)
  • Border
  • Margin

You can set sizes as pixels but it is often recommended to use percentages, as not all browser windows are the same size.

Using "box-sizing: border-box" in CSS is an easy way of calculating box size, but margin is not included. This was developed recently so you need to add browser specific prefixes.

To make div boxes align next to each other use flexbox layout.

Check your HTML and CSS code to see if it’s correct.
